# Primitive: returns

**Doc ID:** gen-dsl/primitive/returns

## Purpose
Declare the output contract: the result must validate against a schema. Two forms — an **inline field block** (the default on-ramp) and a **named-schema reference** (the escape hatch).

## Semantics (normative)
- `returns` accepts EITHER a block (`returns do … end`) OR a name (`returns :Foo` / `returns Foo`), never both — passing both is a compile error.
- **Block form (RED-419):** the `field` declarations compile to a Draft-07 JSON Schema emitted **inline** into the IR under `returnSchema` (stamped `$id: "<ClassName>Output"`). The runtime validates against it directly — **no `contracts.ts` entry, no generated file required to run**.
- **Name form:** emits `returnSchemaId` (a string name-ref); the runtime resolves it against the injected contracts module (the schema lives in hand-written TypeBox). Validated against `contracts.ts` at compile (RED-210).
- The runtime resolves the schema as `ir.returnSchema ?? contractsMod[ir.returnSchemaId]` — inline wins when present; the two are mutually exclusive.
- Validation errors MUST be surfaced in trace.
- The repair loop MUST only change fields implicated by validation errors (default strategy).

## The block form (closed vocabulary)

```ruby
returns do
  field :name, String
  field :count, Integer, optional: true            # → omitted from `required`
  field :ratio, Float
  field :enabled, Boolean
  field :tags, [String]                            # array of scalar
  field :status, String, enum: %w[active archived] # enum on String only
  field :note, String, description: "free text"    # annotation only
  field :details do                                # nested object
    field :summary, String
  end
  field :items, [] do                              # array of object
    field :label, String
  end
end
```

The vocabulary is a **closed enum** (DEC-008) — extended only by explicit decision, each addition additive:

| Ruby in the block | JSON Schema (Draft-07) | TypeBox (generated) |
|---|---|---|
| `String` / `Integer` / `Float` / `Boolean` | `{ "type": "string" \| "integer" \| "number" \| "boolean" }` | `Type.String()` / `Type.Integer()` / `Type.Number()` / `Type.Boolean()` |
| `[T]` (one-element array literal) | `{ "type": "array", "items": <T> }` | `Type.Array(<T>)` |
| `field :x do … end` | nested object with its own `required` + `additionalProperties: false` | `Type.Object({…}, { additionalProperties: false })` |
| `field :x, [] do … end` | array whose `items` is the object schema | `Type.Array(Type.Object(…))` |
| `enum: %w[a b]` (String only) | `{ "type":"string", "enum":["a","b"] }` | `Type.Union([Type.Literal('a'), …])` |
| `optional: true` | field omitted from `required` (not nullable) | `Type.Optional(<T>)` |
| `description: "…"` | adds `"description"` (annotation only; ignored by AJV) | `{ description: '…' }` |

Every object level carries `additionalProperties: false`. Compile errors (each names the field): unknown type token, `enum:` on a non-String, empty block, duplicate field name, `[T]` with ≠1 element type, missing type. **Anything outside this set** (nullable, integer/float enums, `default:`, format/min/max/pattern, tuples, object-unions, open maps) stays on the name form — `returns :Symbol` is the permanent escape hatch.

## Generated TypeScript types

`cambium compile --write` (workspace) regenerates `src/contracts.generated.ts` (TypeBox + inferred types) from every block-form gen, for typed consumer imports (`import { type FooOutput } from './contracts.generated'`). The file is **types-only** — runtime validation uses the inline IR schema, not this file. It opens with the generated-file sentinel (`// @generated by cambium …`); the compiler **hard-errors rather than overwrite** any file lacking that marker (RED-222 family). Never hand-edit it. (Engine mode runs block schemas but does not emit generated TS in 0.8.)

## Examples
```ruby
# Block form (the default — one file, no TypeScript):
returns do
  field :summary, String
  field :key_points, [String]
end

# Name form (the escape hatch — hand-written TypeBox in contracts.ts):
returns AnalysisReport
```

## Failure modes
- **Invalid block (RED-419).** The Ruby collector raises a `CompileError` naming the offending field for: an unknown type token, `enum:` on a non-String, an empty block, a duplicate field name, a `[T]` literal with ≠1 element, or a field with no type.
- **Schema not found at compile time** (RED-210, name form). The Ruby compiler validates `returns <Name>` against the package's `contracts.ts` and fails with a list of available schemas + a "did you mean" suggestion. (The block form needs no contracts lookup — its schema is inline.)
- **Schema not found at runtime** (name form). The runner validates the resolved schema id against its in-memory AJV registry as a final safety net. Block-form gens carry their schema inline, so this never applies.
- **Refusing to overwrite generated contracts (RED-419/RED-222).** `cambium compile --write` hard-errors if `src/contracts.generated.ts` exists without the `@generated by cambium` sentinel on its first line — it will not clobber hand-authored work.
- **Output cannot be repaired into a valid instance.** The repair loop surrenders after `max_attempts` or when no error-count improvement is seen between attempts. See [[C - Repair Loop]].
